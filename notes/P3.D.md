# P3.D — Metric primitives and PerformanceReport

## What was built

`model_crafter.metrics` and `model_crafter.performance`:

* **Discrimination** (`metrics/classification.py`) — `auc`, `gini`, `ks`,
  `cohens_d`, `delong_test`, plus the rich-`__repr__` result dataclasses
  `AUCResult`, `GiniResult`, `KSResult`, `CohensDResult`, `DeLongResult`.
* **Calibration** (`metrics/calibration.py`) — `calibration_curve`,
  `brier_score`, `ece`, `log_loss`, `calibration_slope_intercept`, plus
  the result types `CalibrationCurve`, `BrierResult`, `ECEResult`,
  `LogLossResult`, `CalibrationFit`.
* **Stability** (`metrics/stability.py`) — `psi` (takes two raw
  distributions, not a `Solution`), `PSIResult`.
* **Rank** (`metrics/rank.py`) — `lift_table`, `cumulative_gains`,
  `LiftTable`, `GainsCurve`.
* **PerformanceReport** (`performance/report.py`) — bundles the four
  sub-reports `DiscriminationReport`, `CalibrationReport`,
  `StabilityReport`, `DistributionReport` plus the lift table and gains
  curve. The `__repr__` matches the multi-section block in DESIGN.md
  §3.3 (header / Discrimination / Calibration / Stability /
  Distribution), with the AUC line carrying a DeLong-derived 95% CI.

Each primitive accepts `(sol, data, weights=...)` and returns a frozen
dataclass result with a `__float__` for numeric contexts. Every metric
has a private `_<name>_from_arrays(y, scores, weights)` helper that the
public entry point and the `performance` orchestrator share, so the
"sub-reports match individual primitive calls" acceptance criterion
holds bitwise.

## How the absent logistic loss was handled

P3.A is building `logistic` in parallel and isn't merged on this branch.
Per AGENTS.md P3.D guidance, the tests fit a **squared-error linear
regression** on a Bernoulli target. The fitted `Solution`'s predictions
are roughly in `[0, 1]` (a few may fall slightly outside); calibration
metrics that need probabilities clip internally to `[eps, 1 - eps]`
(default `eps = 1e-12`, documented in the docstrings). AUC, KS,
Cohen's d, lift, gains, and PSI are scale-/range-invariant so the
out-of-range predictions don't affect them.

Where numerical correctness against a reference matters (acceptance
criteria 1–5), the tests hit the `_<name>_from_arrays` helpers directly
with controlled `(y, scores)` arrays — independent of the solver path.

## What's tested

* **AUC** — 100 random trials, AUC matches the Mann-Whitney U identity
  to 1e-9; weighted AUC matches the AUC of a row-replicated sample for
  integer weights (Pepe 2003 §5).
* **KS** — 50 random trials against `scipy.stats.ks_2samp` to 1e-9;
  weighted KS matches replication.
* **PSI** — hand-derived worked example pinned in the docstring with
  expected value `-0.1*log(0.75) + 0.1*log(2.0) ≈ 0.0980829`; matched to
  1e-12. Plus rejects too few bins, rejects non-increasing edges,
  identical distributions give 0, shift growth is monotone.
* **DeLong** — variance of the AUC-difference matches a brute-force
  `O(n²)` reference implementation (DeLong 1988 eq. 6-7) to 1e-6 on a
  fixed seed (`np.random.default_rng(42)`, n=200). The fast version is
  Sun & Xu (2014) Algorithm 1.
* **Calibration slope/intercept** — recovers `(1, 0)` on perfectly
  calibrated synthetic data within sampling SE, and matches statsmodels'
  `Logit` MLE to 1e-5 on `n=5000`.
* **ECE** — hand-derived 5-bin worked example pinned to 0.42 exactly.
* **PerformanceReport** — every sub-report value bitwise-equals the
  corresponding standalone primitive call (acceptance #6). The repr
  contains every section label from DESIGN.md §3.3.

## Reference sources for `delong_test`

* **DeLong, DeLong & Clarke-Pearson (1988)** — the original
  structural-component variance formula, Biometrics 44(3): 837-845.
* **Sun & Xu (2014)** — *Fast implementation of DeLong's algorithm for
  comparing the areas under correlated receiver operating characteristic
  curves*, IEEE Signal Processing Letters 21(11): 1389-1393. This is the
  `O((m+n) log(m+n))` midrank reformulation we implement.
* **Test reference** — a brute-force `O(n²)` implementation of the
  structural components V10 and V01 (DeLong eq. 6-7) is computed
  independently inside `test_metrics.py::_delong_brute_force` and
  agrees with the fast version to 1e-6 at the pinned seed.

The AGENTS.md note suggests `pROC::roc.test` (R) as the canonical
reference but we don't have R available; the brute-force-from-paper
implementation in the test file is the equivalent ground truth.

## Open / for the integration agent

1. **Weighted DeLong is not implemented.** The unweighted DeLong variance
   is exposed via `delong_test`; passing non-uniform weights raises
   `NotImplementedError` and points at Pepe (2004) §5.2. The
   `PerformanceReport`'s AUC CI is computed via the one-sample DeLong
   variance and falls back to `None` when weights are non-uniform. This
   matches the AGENTS.md contract (DeLong is unweighted in v0).

2. **Public API re-exports.** Per AGENTS.md, P3.D does not edit
   `model_crafter/__init__.py` — that lives with P3.INTEG. To use the new
   API today, the integration agent should add:

   ```python
   from model_crafter.metrics import (
       auc, gini, ks, cohens_d, delong_test,
       brier_score, calibration_curve, calibration_slope_intercept,
       ece, log_loss, psi, lift_table, cumulative_gains,
   )
   from model_crafter.performance import performance
   ```

   to `model_crafter/__init__.py`'s exports.

3. **PSI signature.** AGENTS.md pins `psi(reference, current, bins=10)`.
   I added two optional kwargs (`weights_reference`, `weights_current`,
   `eps`) but the positional signature is unchanged. Mention to flag if
   the integration agent prefers a tighter surface.

4. **AUC CI in the header repr.** DESIGN.md §3.3's repr example shows the
   AUC line with a DeLong 95% CI: `(95% CI: 0.8050 – 0.8197, DeLong)`.
   That CI is computed by the orchestrator and stored on
   `DiscriminationReport.auc_ci` / `auc_se`. When weights are non-uniform
   we omit the CI (because weighted DeLong is out of scope).

5. **No runtime statsmodels dep.** `calibration_slope_intercept`
   implements its own logistic MLE via `scipy.optimize.minimize` with
   `np.logaddexp` for numerical stability and `scipy.special.expit` for
   the sigmoid — statsmodels is only imported in the test as a
   cross-check (DESIGN.md §9.10).
