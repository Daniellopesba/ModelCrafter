## Open items / contract notes for later phases

- P2.A widened the `Penalty` protocol to require `prox`. `NoPenalty` got
  an identity `prox` to satisfy the protocol. If a later phase wants to
  add penalty types that genuinely don't support prox (rare — most
  penalties used in convex optimisation do), revisit whether `prox`
  should be on the protocol vs a per-type optional method.
- P2.C's coordinate descent standardises X internally and de-standardises
  coefficients on return (glmnet convention). This is the convention
  Phase 3+ tuning/CV code must respect — flagged in `notes/P2.C.md`.
- P2.B's ridge does not return closed-form standard errors
  (`coefficient_se=None`); the hat-matrix derivation exists but was
  deferred. Bootstrap (Phase 3) is the documented path to coefficient
  CIs for any penalised fit.
- `model_crafter/validation/` now exists (currently just `lambda_path`).
  Phase 3 will add CV runners, tuning, bootstrap into this subpackage.
