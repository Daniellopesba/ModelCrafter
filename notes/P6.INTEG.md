## v0 is complete

DESIGN.md §8 Phase 6 acceptance is met: a segmented logistic regression
with WoE features produces per-segment Solutions, AssumptionReports,
BootstrappedSolutions, and PerformanceReports from a single declarative
spec.

The §10 north-star example now runs end-to-end:

```python
import model_crafter as mc

# Build a WoE-encoded logistic spec, segmented by product.
base = mc.linear(
    target="default_12m",
    features=(
        mc.woe("income", bins=mc.monotonic(min_bin_size=0.05))
        + mc.woe("age",   bins=mc.monotonic())
        + mc.woe("region", bins=mc.categorical(group_rare=0.01))
    ),
    loss=mc.logistic,
    penalty=mc.l2(0.1),
)
spec = mc.segmented(by="product", base=base)

# Solve once. Per-segment fits fall out automatically.
sol = mc.solve(spec, df)
for product, sub_sol in sol.segments.items():
    print(sub_sol.assumptions)   # per-segment AssumptionReport

# Walk-forward CV, bootstrap, performance — all on a single Solution
# or the underlying base spec.
splitter = mc.expanding_window(time_col="origination_dt",
                               n_folds=5, horizon="90D", gap="365D")
perf_t = mc.performance_over_time(sol["card"], df.query("product=='card'"),
                                  splitter)
perf_seg = mc.performance_by_segment(sol["card"], df.query("product=='card'"),
                                     by="vintage_quarter")
```

## Open items (post-v0)

- **BCa CIs** in `BootstrappedSolution.coefficient_ci(method="bca")` still
  stubbed (P3.C deferral).
- **Weighted DeLong** still raises `NotImplementedError` (P3.D deferral).
- **`label_horizon` on `LogisticLoss`** is unset — `cross_validate`'s
  refusal-on-zero-gap check has no horizon to compare against until a
  later phase adds the configuration knob.
- **Auto-fit of stateful WoE terms** inside `mc.solve` is wired only
  through `solve_segmented` (which calls `fit_binnings` transparently).
  For non-segmented WoE specs the user still has to call
  `fit_binnings(spec, data)` first. A unifying pass would move that
  call into the top-level `solve()` flow.
- **`ExpandedTerm.state` field** for basis-term knot round-trip
  remains a P4-era hack; P4.A's terms stash knots inside the term.
  Clean fix is a Phase-7 refactor.
- **Diagnostics signature** in DESIGN.md §5 shows
  `diagnostics(sol) -> Diagnostics`, but the implementation requires
  `diagnostics(sol, data)` because `Solution` doesn't persist `X`.
  Either persist a (sparse) `X` on `Solution`, or update DESIGN.md.

These are all v0+ work — none block the v0 acceptance.
