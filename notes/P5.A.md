# P5.A — Temporal performance

## What I built

`model_crafter/performance/over_time.py` ships the `TemporalPerformanceReport`
frozen dataclass and the `performance_over_time(sol, data, splitter, *,
reference=None, weights=None)` constructor. The signature matches the contract
pinned in AGENTS.md / DESIGN.md §3.3. Behaviour:

- Iterates `splitter.split(data)` and keeps only the *validation* slice from
  each `(train, valid)` pair — the model is fixed and is not refit.
- For every non-empty validation window, calls `mc.performance(sol, valid,
  weights=weights, reference=reference)`. The same `reference` anchors PSI
  drift across all windows (DESIGN.md §3.3 deployment-monitoring framing).
- Builds a `summary` `DataFrame` indexed by the *midpoint* of each
  validation window's time column (same convention as P3.B's `over_time`),
  with columns `n_obs`, `n_events`, `auc`, `ks`, `brier`, `log_loss`,
  `mean_p`, plus `psi` whenever `reference` is supplied.
- `TemporalPerformanceReport.__repr__` prints the summary frame followed by
  one ISO-timestamped bullet per window (AUC, KS, Brier, optionally PSI).
- `plot()` is the lazy-matplotlib helper: imports `matplotlib.pyplot` inside
  the method and raises a clear `ImportError` if it's missing. Plots AUC and
  KS on the primary axis, PSI on a secondary axis when present (with a
  0.25 reference line).

## What's tested

`tests/test_performance_over_time.py` (11 tests). The headline acceptance
test (`test_performance_over_time_reproduces_known_AUC_drift`) uses a
deterministic interleaving panel: within each block of 100 positives and
100 negatives, positives occupy the middle `m=100` score slots and
negatives the lowest `k_i` and highest `m - k_i`. Mann-Whitney gives
`AUC = k_i / m` exactly. Picking `k_i ∈ {90, 70, 50, 30, 10}` drifts AUC
from 0.9 to 0.1 across five windows. A no-intercept OLS fit with
`target == score` produces an identity predictor (`coef = 1`), so
`mc.predict(sol, panel)` equals the engineered scores to machine
precision and the reported AUC matches the closed-form value. Tolerance
in `assert_allclose` is `atol=1e-4` per the acceptance criterion; the
actual error is bit-exact.

Other tests cover: returned dataclass type, midpoint indexing, summary
columns with/without reference, PSI presence when reference is given,
weights resolved per window (uniform `w=1` matches unweighted), `n_obs`
and `n_events` correctness, `__repr__` layout, frozen-dataclass mutation
rejection, lazy matplotlib `plot()` returning a `Figure` under the Agg
backend, and graceful handling of splitters that yield no validation
windows.

Verification (fresh output): `ruff check .` clean, `pyright` reports
`0 errors, 0 warnings, 0 informations`, `pytest -q` reports
`605 passed` (594 baseline + 11 new).

## Open

- The splitter used in the acceptance test is `mc.purged_kfold` because the
  alternative (`rolling_window`) excludes rows whose timestamps land
  exactly on its half-open boundaries; `purged_kfold`'s equal-count
  bucketing is the cleanest way to land each engineered block in its own
  validation slice. The implementation under test is splitter-agnostic
  — any object exposing `time_col` and `.split(df)` works. I noted this
  in the test's docstring rather than touching `rolling_window`, which is
  outside this workspace.
