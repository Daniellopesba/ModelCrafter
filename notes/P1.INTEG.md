## Integration fixes against the merged worktrees

- `loss.py`: `squared_error.assumptions` now declares the four
  classical INFO checks (`ResidualNormality`, `Homoscedasticity`,
  `Independence`, `LowVIF`) alongside `FullRankDesign`, matching
  DESIGN.md §4.3's catalogue. Without this, the framework had no
  INFO assumptions to run when `classical_inference=True`.
- `solve/__init__.py`: the post-fit `run_assumptions` call now passes a
  lightweight `SimpleNamespace(coefficients, coefficient_se,
  design_columns)` as `solution=...` so classical checks needing
  residuals (Homoscedasticity, Independence) actually run.
- `solve/__init__.py`: dropped the `design=`/`offending_columns=`
  kwargs (the framework's `FullRankDesign.check` materialises its own
  design); kept the explicit `AssumptionError` raise so warn/ignore
  modes still abort on singular X.
- `pyrightconfig.json`: added `venvPath: "."` and `venv: ".venv"` so
  bare `pyright` picks up third-party packages without `--pythonpath`.
- Static-typing nits: `pd.Index([...])` instead of `list[str]` for
  `columns=`; `cast(pd.Series, ...)` around `coefs.mean(axis=0)`;
  `.loc[mask]` instead of `df[mask]` in the integration tests.

## Open items for later phases

- DESIGN.md §4.1 says "classical inferential tests are not declared by
  losses" but §4.3's catalogue lists them as declared by
  `SquaredErrorLoss`. The catalogue won. If we move to per-call
  injection later, revisit `loss.py`.
- Stability checks (`CoefficientStability`, `PredictiveStability`)
  remain CV-stubbed until P3.B; the CV-backed paths read a best-effort
  `cv.coefficients`/`cv.metrics`/`cv.fold_results` shape and may need
  a small adapter once P3.B pins the real `CVResult` interface.
- `model_crafter.assumptions._common.materialise_design` duplicates the
  design-matrix build inside `model_crafter/_internal/design.py`. Once
  basis-expansion terms arrive (Phase 4), collapse the two so VIF /
  rank checks honour the same expansion.
