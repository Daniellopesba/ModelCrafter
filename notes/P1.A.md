# Task P1.A — Terms, LinearSpec, OLS solver, Solution

## What got built

The Phase 1 numerical core. `model_crafter.terms.base` ships the `Term`
protocol, `RawTerm` (the string-promoted column reference), `TermSum`
(flattened, associative `+`), and the `_promote` / `_normalize_features`
helpers `linear()` calls. `model_crafter.loss` ships the `Loss` protocol
and the `squared_error` singleton, which declares its `assumptions`
(currently just `FullRankDesign`) via a lazy-import property so the module
stays importable in isolation from P1.B. `model_crafter.penalty` ships the
`Penalty` protocol and `NoPenalty`; `+` between a `Penalty` and a `Term`
raises a `TypeError` pointing at `features=` vs `penalty=`.
`model_crafter.spec.linear(...)` returns a frozen, slotted, hashable
`LinearSpec`; `model_crafter.solution.Solution` is its post-solve
counterpart with the assumption report attached. Numerical work lives in
`model_crafter/_internal/design.py` (intercept + per-term expansion + a
no-silent-NaN guard) and `model_crafter/_internal/linalg.py` (pivoted-QR
rank check that names offending columns, and a QR-direct OLS/WLS solve
matching R's `lm` to machine precision). The dispatch entry point
`model_crafter.solve.solve` and `predict` live in
`model_crafter/solve/__init__.py`; solvers self-register through
`model_crafter/solve/_registry.py` so Phase 2's ridge and lasso modules
can ship in parallel without editing the dispatch file. `solve()`
materialises the design, runs `FullRankDesign` *before* the numerical
solve (raises `AssumptionError` naming offending columns when violated),
runs the solver, then calls P1.B's `run_assumptions` for the post-fit pass
and attaches the resulting `AssumptionReport` to the `Solution`.

## What's tested

`tests/test_terms.py` (18), `tests/test_loss.py` (6),
`tests/test_penalty.py` (5), `tests/test_spec.py` (13),
`tests/test_solution.py` (3) pin the smallest behaviours of each module.
`tests/test_ols.py` (13) carries the acceptance criteria: the ESL §3.2
prostate fit (with the dataset committed to `tests/data/prostate.csv` and
the ESL preprocessing — standardize on the full 97 rows, fit on the 67
training rows) matches **statsmodels.OLS to ~1e-16** on every quantity the
spec lists (coefficients, SEs, residual std error, R²) — comfortably
inside the 1e-8 acceptance tolerance — and the ESL Table 3.2 printed
values match to two-decimal precision; rank-deficient designs (both an
exact-collinearity case and an intercept-vs-constant-column case) raise
`AssumptionError` with the offending column named in the message;
`mc.predict` returns a `pd.Series` aligned to `new_data.index` (custom
index, name preserved) and matches the manual `β₀ + Σ βⱼ xⱼ` computation
exactly; weighted OLS via `weights="col"` and `weights=array` both match
`statsmodels.WLS` to 1e-10. Lint (`ruff check .`), type-check (`pyright`,
0 errors) and the full pytest run (`58 passed`) are all green.

## Stubs / mocks / open questions

* P1.A treats `model_crafter.assumptions` as opaque. `tests/conftest.py`
  installs a minimal in-memory stub of P1.B's public interface
  (`AssumptionError`, `AssumptionReport`, `CheckResult`, `Severity`,
  `FullRankDesign`, `run_assumptions`, `check_assumptions`) into
  `sys.modules` so my tests can run in this worktree. The stub honours the
  contract from AGENTS.md (P1.B section) and the call sites in
  `model_crafter/solve/__init__.py` pass `design=` and
  `offending_columns=` kwargs into `run_assumptions` (gracefully falling
  back to raising directly if P1.B's signature does not accept them yet).
  **Integration step:** delete `tests/conftest.py` once P1.B's real module
  is on the branch, and replace the four
  `# pyright: ignore[reportMissingImports]` comments scattered across
  `model_crafter/loss.py`, `model_crafter/solution.py`,
  `model_crafter/solve/__init__.py` and `tests/test_ols.py` /
  `tests/test_solution.py`.

* `Solution.assumptions: AssumptionReport` uses a `TYPE_CHECKING`-guarded
  import; the dataclass field works at runtime with any object. If P1.B's
  `AssumptionReport` ends up with a different name or shape, only the
  guarded import + the conftest stub need to follow.

* Workspace expansion: I added `tests/test_loss.py`, `tests/test_penalty.py`,
  `tests/test_spec.py`, `tests/test_solution.py` in addition to the two
  acceptance-test files the task listed. Each maps to a module I own per
  the convention `tests/<owned-module-name>.py`. Flagging here in case
  the integration agent wants to consolidate.

* `_normalize_weights` is enforced consistently across `loss.py` and
  `solve/__init__.py`. They both raise on NaN/inf, negatives, or all-zero
  weights — per DESIGN.md §9.6 + §9.8.

* The dispatch registry is keyed on **classes**
  `(type(loss), type(penalty))` not on instances; this matches how Phase 2
  expects to register `(_SquaredErrorLoss, L2Penalty)`. The OLS solver
  self-registers in `solve/ols.py`; `solve/__init__.py` imports it for
  the side-effect.

* `model_crafter/__init__.py` was untouched (P1.INTEG owns the public
  re-export list per AGENTS.md).

* No work was done outside the P1.A workspace except (a) committing the
  prostate CSV under `tests/data/` and (b) the conftest stub described
  above. `model_crafter/assumptions/` and `tests/test_assumptions*` are
  exactly as P0 left them (empty).
