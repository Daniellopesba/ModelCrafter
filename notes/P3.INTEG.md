## Open items / contract notes for later phases

- `LogisticLoss.label_horizon` is unset. The `cross_validate` machinery
  (P3.B) already calls `getattr(spec.loss, "label_horizon", None)` and
  refuses to run a zero-gap splitter against a label-horizon loss â€” it
  just has no horizon to compare against today. A later phase
  (Phase 4 or a dedicated config) should expose a way to set it per
  spec (`mc.linear(..., label_horizon="365D")` is the natural shape).
  Until then, callers can wrap the loss themselves.
- Stability checks (`CoefficientStability`, `PredictiveStability`) are
  declared on `squared_error.assumptions` and `LogisticLoss.assumptions`
  but only run when `cv` is passed to `run_assumptions`. The
  `cross_validate` runner returns a `CVResult` with `fold_results` that
  P1.B's stability code already knows how to read, but the assumption
  framework is not yet auto-invoked from `cross_validate`. P3.B's note
  flags this as a P3.INTEG-or-later wiring; deferred so the
  responsibility stays in one place rather than spreading across two
  modules.
- BCa CIs in `BootstrappedSolution.coefficient_ci(method="bca")` raise
  `NotImplementedError`. Percentile CIs work and cover the acceptance
  test; BCa is a documented v0 deferral (`notes/P3.C.md`).
- DeLong with non-uniform weights is unimplemented; `delong_test` with
  weights raises `NotImplementedError`. PerformanceReport's AUC CI
  falls back to `None` when weights are non-uniform.
- The `slow` marker now exists; the heavy bootstrap tests run as part
  of the default suite (still under a minute). Run `pytest -m 'not slow'`
  to skip them.
